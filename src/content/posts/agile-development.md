## 애자일이란?

애자일(Agile)은 소프트웨어 개발에서 빠른 반복과 지속적인 개선을 강조하는 방법론입니다. 2001년 애자일 선언문(Agile Manifesto)이 발표되면서 소프트웨어 개발 패러다임에 큰 변화를 가져왔습니다.

## 애자일의 4가지 핵심 가치

1. **프로세스와 도구보다 개인과 상호작용**을 중시합니다.
2. **포괄적인 문서보다 작동하는 소프트웨어**를 중시합니다.
3. **계약 협상보다 고객과의 협력**을 중시합니다.
4. **계획을 따르기보다 변화에 대응**하는 것을 중시합니다.

## 주요 실천 방법

### 스프린트(Sprint)

짧은 주기(보통 1-4주)로 개발을 반복하며, 각 스프린트마다 작동 가능한 제품을 만들어냅니다.

```plaintext
계획 → 개발 → 테스트 → 리뷰 → 회고
```

### 데일리 스탠드업

매일 짧은 시간(15분) 동안 팀원들이 모여 진행 상황을 공유합니다:

- 어제 무엇을 했는가?
- 오늘 무엇을 할 것인가?
- 어떤 장애물이 있는가?

### 회고(Retrospective)

스프린트가 끝날 때마다 팀이 모여 다음을 논의합니다:

- 잘된 점
- 개선할 점
- 다음 스프린트에서 시도할 것

## 애자일의 장점

> [!TIP]
> 애자일은 변화하는 요구사항에 빠르게 대응할 수 있어, 고객 만족도를 높이고 위험을 줄일 수 있습니다.

- **빠른 피드백**: 짧은 주기로 고객에게 결과물을 보여주고 피드백을 받습니다.
- **높은 유연성**: 요구사항 변경에 유연하게 대응할 수 있습니다.
- **팀 협업 강화**: 지속적인 소통으로 팀워크가 향상됩니다.
- **위험 감소**: 작은 단위로 개발하고 검증하여 큰 실패를 방지합니다.

## 대표적인 애자일 프레임워크

### 스크럼(Scrum)

가장 널리 사용되는 애자일 프레임워크로, 명확한 역할과 이벤트를 정의합니다.

- **Product Owner**: 제품의 비전과 우선순위 관리
- **Scrum Master**: 팀이 스크럼 프로세스를 잘 따르도록 지원
- **Development Team**: 실제 개발을 수행하는 팀원들

### 칸반(Kanban)

작업 흐름을 시각화하고 진행 중인 작업을 제한하여 효율성을 높입니다.

```
To Do → In Progress → Review → Done
```

## 마무리

애자일 방법론은 단순한 프로세스가 아니라 **마인드셋의 변화**입니다. 완벽한 계획보다는 빠른 실행과 개선, 문서보다는 소통과 협력을 중시하는 문화를 만들어가는 것이 중요합니다.
